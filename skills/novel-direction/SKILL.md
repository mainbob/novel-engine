# /novel-direction

> 在写作过程中调整方向——分析影响，更新大纲和规则。

---

## 触发方式

```
/novel-direction                       # 开始方向调整对话
```

## 前置条件

- 项目已有至少一章已写内容
- blueprint.json 存在

## 执行流程

1. 接收用户的调整请求
2. 调用 Direction Advisor Agent 分析影响
3. 展示影响分析报告
4. 用户确认后执行调整：
   - 更新大纲
   - 更新规则文档（如果需要）
   - 更新蓝图（如果需要）
   - 记录方向调整事件到 state.json

## 输出

- 更新后的大纲、规则、蓝图
- `state.json` 中的方向调整记录
