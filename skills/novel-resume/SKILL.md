# /novel-resume

> 中断恢复——自动识别断点，恢复到上次中断的位置。

---

## 触发方式

```
/novel-resume                          # 自动检测断点并恢复
```

## 前置条件

- 项目目录存在

## 执行流程

1. 读取 state.json → 最后完成的章节号
2. 检查是否有未完成的写作流程：
   - 有 context-{N}.md 但没有对应章节文件 → 写作中断
   - 有章节文件但没有 review-{N}.json → 审查中断
   - 有 review 但没有更新 state → 数据沉淀中断
3. 从断点恢复对应步骤

## 输出

恢复执行，从断点继续。
