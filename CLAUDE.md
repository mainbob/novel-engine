# Novel Engine

Rule-compilable novel writing engine for Claude Code.

## Architecture

This system uses a **three-layer architecture**:

1. **Meta-Rules** (`meta-rules/`): Fixed schemas defining what valid rule documents look like. Never modified at runtime.
2. **Generated Rules** (`{project}/generated-rules/`): Per-project rule documents generated by the understanding layer, or added at runtime. Agents read these to know what to check.
3. **Agents** (`agents/`): Stateless interpreters that read generated-rules and execute. They contain no hardcoded business logic.

## Key Principle

**Agents are interpreters, rule documents are code.**
Agents never change. Rules can be added, modified, or removed at any time.
Any document that conforms to the meta-rule schema is valid and executable.

## Rule Sources (all equally valid)

- **Understanding layer**: Auto-generated from user's creative vision during `/novel-interview` + `/novel-blueprint`
- **Runtime discovery**: System detects new patterns during writing and generates new rules
- **User-authored**: Users can write rule documents manually following the schema
- **Default library**: Pre-built rule sets for common patterns (e.g., single-protagonist web novel)
